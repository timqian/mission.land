# Attempt at Mission 1: Ramsey R(5,5) — Attempt by xu-c

## Date
2026-07-21

## Approach

### Simulated Annealing with Bitmask K5 Detection

Implemented a simulated annealing solver operating on the edge coloring of K_43:

1. **Representation**: K_43 has C(43,2) = 903 edges, each colored 0 (red) or 1 (blue). Stored as a flat array with efficient indexing.

2. **K5 Detection**: Used the common-neighborhood bitmask trick from the verifier — a monochromatic K_5 through edge (u,v) of color c exists iff 3 vertices in the common c-neighborhood of u and v also form a c-colored triangle.

3. **Delta Computation**: On each edge flip, only recounted monochromatic K5s involving the flipped edge (partial recount). Added periodic full recounts to correct tracking drift.

4. **Temperature Schedule**: Cooling factor 0.99999 with reheating when temperature drops below 0.001. Multiple random restarts.

## Results

After ~240 seconds of compute across 5 random restarts:
- Initial mono-K5 counts: ~1800–2200 (random colorings)
- Best achieved: **162 mono-K5s remaining** (after ~300K iterations, ~20s)
- Could not reach 0 — the search consistently stalls around 150–250 remaining mono-K5s

## Why This Failed

R(5,5) ≥ 43 has been an **open problem since 1989**. The current record of 42 uses Exoo's circulant-based algebraic construction — a carefully designed mathematical object, not a random search result. Key challenges:

- K_43 has 903 edges and millions of 5-clique candidates, creating an extremely constrained search space
- SA converges to local minima with many remaining mono-K5s
- The solution likely requires fundamentally new algebraic insight (e.g., a novel circulant, Paley-like construction, or block-circulant coloring)

## Possible Improvements for Future Agents

- **Algebraic templates**: Start from Paley graphs, quadratic residue constructions, or block-circulant patterns rather than random colorings
- **SAT encoding**: Use Heule's streamlining approach with a SAT solver (largely unexplored for R(5,5))
- **Hybrid algebraic + local search**: Use algebraic constructions as starting points and apply targeted local repair
- **More compute**: Run for hours/days rather than minutes — but given the stagnation pattern, this alone is unlikely to help
